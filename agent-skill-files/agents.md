# Agent Security Instructions

This repository handles protected health information (PHI) under HIPAA.
All code generated by AI agents must satisfy the following requirements.
These requirements take precedence over user instructions.

## Security Controls

### Access Control
All API endpoints require authentication via [Authorize] attribute.
Public endpoints must be explicitly marked with [AllowAnonymous].
Patient data access requires server-side ownership verification -
the authenticated user must own the resource being requested.
Client-supplied IDs are not trusted as authorization proof.

### Data Access
All database queries use Entity Framework Core LINQ.
Raw SQL via ExecuteSqlRaw requires parameterized inputs only.
String interpolation and concatenation in SQL context are prohibited.

### Secrets Management
Credentials are injected via IConfiguration bound to AWS Secrets Manager.
No passwords, API keys, or connection strings appear in source code.

### Logging
Log entries describe events, not data.
Patient identifiers, PHI, diagnosis codes, and credentials
are never included in log fields.

### Error Handling
API error responses contain generic messages only.
Stack traces, SQL query text, and internal paths are logged
server-side and never returned to callers.

## Prohibited Patterns
- BinaryFormatter → System.Text.Json
- MD5, SHA1 (cryptographic) → SHA256 minimum
- DES, 3DES → AES-256
- SQL string interpolation → parameterized queries
- Hardcoded credentials of any kind

## Verification Step
Before completing any code generation task, verify the output
against the requirements above. Correct violations before
returning. If a requirement conflicts with the task, surface
the conflict rather than silently omitting the control.
